# 小红书笔记内容脚本生成器

## 1. 核心目标

本技能旨在辅助将一个技术主题或一篇文章，解构成一份“内容脚本”。用户可以根据这份脚本，去制作小红书的图片轮播笔记。

最终输出的是一份 Markdown 文件，其中包含了为每张图片设计的文字内容，以及笔记的正文描述区文案。

## 2. 工作流程

由于小红书是高度视觉化的平台，AI 无法直接生成图片。因此，我们的核心任务是生成制作图片所需的**文字内容**。请严格遵循以下步骤：

### 第 1 步：分析源内容并解构

- 读取用户提供的文章或主题，理解其核心知识点。
- **解构思想**: 将整个知识体系拆解成 7-9 个独立的、可在一张图内讲清楚的“知识卡片”。第一个是封面，最后一个是 CTA。
- **与用户确认**: 向用户展示你的解构大纲（例如：“我将为您规划一个 9 图笔记，大纲如下：1. 封面；2. 知识点A；3. 知识点B...；9. 总结与关注。您看可以吗？”）。

### 第 2 步：设计图片文字脚本（核心）

依次为每一张图片（卡片）生成文字内容。

- **图片 1: 封面**
    - **目标**: 设计封面图的标题和核心卖点。
    - **操作**: 创作一个充满 Emoji、结果导向、有吸引力的标题。
    - **示例**: `前端求职千万别错过✨| 3 个 React 简历加分项目`

- **图片 2-8: 核心内容**
    - **目标**: 为每个知识卡片撰写简短、精炼的文字。
    - **一图一事**: 确保每张卡片的文字只聚焦一个核心知识点或一个操作步骤。
    - **极简代码**: 如果需要展示代码，**只保留 1-2 行最关键的部分**作为示意，并配上通俗易懂的解释。
    - **多用“结果”**: 引导用户多使用效果截图，而非代码截图。

- **图片 9: 行动号召 (CTA)**
    - **目标**: 设计最后一张引导图的文字。
    - **操作**: 明确引导用户“点赞收藏，以备不时之需”或“关注我，学习更多...”，并预留出放置公众号二维码或个人联系方式的位置。

### 第 3 步：撰写正文描述区

- **目标**: 撰写图片下方的补充说明文字。
- **语言风格**: 使用极度口语化的风格，像和朋友分享一样，可以适当加入一些个人感受或“踩坑”经历。
- **内容**: 简要介绍笔记的核心内容，并引导用户看图。
- **Hashtag**: 在文案的最后，生成一串（5-10个）高度相关的 Hashtag，这是获取流量的关键。例如：`#前端开发 #React #求职 #简历 #学习`。

### 第 4 步：整合并调用 `code_docs-creator` 输出

- **整合内容**: 将以上所有产出（标题、每张图片的文字脚本、正文描述区文案）整合到一个 Markdown 文件中，形成一份完整的“发布脚本”。
- **添加日期**: 在文档开头，明确标注“生成日期”，并使用当前系统日期。
- **确定文件名**: 生成一个符合规范的中文文件名，例如 `小红书笔记脚本_{主题}.md`。
- **调用 `code_docs-creator`**:
    - 激活 `code_docs-creator` 技能。
    - 指示 `code_docs-creator` 将整合后的“脚本”内容，以上述文件名，写入到默认的 `docs/` 目录下。
- **最终确认**: 告诉用户“内容脚本”已生成，并提供文件路径。提醒用户根据这份脚本去制作图片。
