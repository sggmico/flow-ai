# 小红书笔记发布核心规范

本规范旨在将一个技术知识点或文章，转化为适合在小红书平台发布的、以图片轮播为核心的笔记“脚本”。

## 核心原则

- **内容即图片**: 核心内容必须以文字形式设计在图片上。正文描述区只是补充。
- **视觉驱动**: 封面图是笔记的生命线，必须在 2 秒内抓住用户注意力。
- **结果导向**: 多展示“做完之后”的酷炫效果，少讲枯燥的理论。
- **“种草”与“收藏”**: 核心目标是让用户觉得“这个我用得上”，从而收藏笔记并关注博主。

## 格式与风格要点

### 1. 整体结构

- **图片轮播**: 7-9 张方形图片构成一篇笔记的核心。
- **内容分配**: 每张图片只讲一个核心知识点，文字要少而精。

### 2. 标题

- **风格**: 强结果导向，突出价值，多用 Emoji ✨, 🚀, 💡 等。
- **结构**: 强调“教程”、“保姆级”、“加分项”，让用户有获得感。
- **示例**: `前端工程师简历加分项✨| 3步让你的网页秒变APP`

### 3. 图片内容设计 (脚本核心)

- **封面 (图 1)**:
    - 信息量饱满，标题醒目，配色抓眼。
    - 必须清晰地告诉用户“你能从这篇笔记里学到什么”。
- **内容页 (图 2-8)**:
    - **一图一事**: 每张图只讲一个步骤或一个知识点。
    - **代码点缀**: **绝不**贴大段代码。只放 1-2 行最关键的代码作为视觉元素，并配上大白话解释。
    - **多用截图**: 多放展示最终效果的截图，给用户最直接的视觉冲击。
- **结尾页 (图 9)**:
    - 总结全文，或设置强行动号召 (CTA)。
    - 引导用户“点赞收藏，以备不时之需”。

### 4. 正文描述区

- **作用**: 对图片内容进行补充说明，或分享一些“踩坑”心得。
- **语言**: 极度口语化，像和姐妹聊天一样。
- **结尾**: **必须** 带上大量相关的 Hashtag，例如 `#前端` `#求职` `#简历技巧` `#程序员` 等。这是流量的关键。
