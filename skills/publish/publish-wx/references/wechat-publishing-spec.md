# 微信公众号文章发布规范

本规范旨在将标准 Markdown 技术文章转化为适合微信公众号平台发布的格式，核心是优化移动端阅读体验和社交传播性。

## 核心原则

- **读者导向**: 内容需对技术爱好者、初中级开发者友好，易于理解和“扫读”。
- **视觉优先**: 移动端阅读以视觉驱动，图片、卡片、图表比大段文字更具吸引力。
- **社交传播**: 所有设计都应服务于“分享”这一核心目标。

## 格式与风格要点

### 1. 标题

- **风格**: 感性、有吸引力，能引发好奇心或戳中痛点。
- **结构**: “主标题 (吸引人) + 副标题 (解释性)”。
- **示例**: `前端工程化新思路：让你的 Web 应用像 App 一样丝滑`

### 2. 正文

- **语言**: 口语化、亲切，多用短句和短段落 (一两句话一段)。
- **排版**:
    - **重点**: 使用 **加粗**、引用、emoji (💡, ✅, ❌) 等方式突出关键信息。
    - **Emoji 数量控制**: 仅在关键点少量使用，避免密集或连续出现，保持整体专业严肃。
    - **代码块**: **必须** 使用 `carbon.now.sh` 或类似工具生成美观的**图片**。绝不直接粘贴代码文本。
    - **逻辑/观点**: 复杂逻辑、核心观点、关键步骤应制作成**图片卡片**或**流程图**，方便读者快速理解和记忆。

### 3. 摘要

- **目的**: 吸引用户在朋友圈和群聊中点击。必须精心撰写，突出文章亮点和价值。

### 4. 行动号召 (CTA)

- **目标**: 促进社交传播，并引导关注。
- **措施**:
    - 文末引导读者“点赞”、“在看”、“分享”。
    - 设置互动话题，鼓励评论。
    - 通过“阅读原文”链接到 GitHub 博客或其他深度阅读材料。
    - 引导关注公众号。
