# GitHub 博客文章发布器

基于参考文章生成符合 GitHub Pages 博客规范的技术文章。

**完整规范**: 生成前必读 `references/github-blog-spec.md`

## 工作流程

```
获取参考文章 → 读取规范 → 分析结构 → 生成文章 → 验证清单 → 输出
```

### 步骤 1: 获取输入

要求用户提供:
- 参考文章路径（必需）
- 目标主题（可选，默认沿用原文）
 - 输出目录固定为参考文章同级目录（无需用户提供）

### 步骤 2: 读取规范

读取 `references/github-blog-spec.md`，掌握:
- 受众定位与消费习惯
- 语言风格要求（严谨、深入、逻辑清晰）
- 代码注释规范（解释设计意图）
- 可信度建设要求（来源链接）

### 步骤 3: 分析参考文章

快速提取（不逐行复制）:
- 核心主题与结构框架
- 技术要点（3-5 个）
- 可复用的代码示例模式

### 步骤 4: 生成文章

按规范生成，确保:
- Front-matter 完整（title, date, tags, description）
- 代码关键行有**设计意图**注释
- 引用数据附来源链接
- 包含延伸阅读和 CTA

### 步骤 5: 验证清单

对照 `references/github-blog-spec.md` 第 5 节验证:
- [ ] Front-matter 完整
- [ ] 链接有效
- [ ] 代码注释清晰
- [ ] 引用有来源
- [ ] 标题适合 SEO

### 步骤 6: 输出

调用 `code-docs-creator` 规范:
- 格式: Markdown
- 命名: `[原文文件名]-github版.md`
- 语言: 中文优先
- 标点: ASCII
- 日期: 系统实际日期

## Token 优化

- 规范文档按需加载，不重复读取
- 分析阶段只提取要点
- 单次生成，验证后修改
