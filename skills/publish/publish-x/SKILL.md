# X (Twitter) 推文脚本生成器

## 1. 核心目标

本技能旨在辅助将一个技术主题或一篇文章，转化为一份适合在 X (Twitter) 发布的“推文线索 (Thread)”脚本。

最终输出是一份 Markdown 文件，其中包含了构成整个 Thread 的每一条推文的独立内容。

## 2. 工作流程

Twitter 的信息流节奏极快，内容的结构和形式至关重要。请严格遵循以下步骤生成推文脚本。

### 第 1 步：分析源内容并确定策略

- 读取用户提供的文章或主题，提炼其核心论点和最吸引人的亮点。
- **判断形式**:
    - 如果内容足够简短，可以作为单条推文发布，则直接进入第 2 步，然后跳到第 5 步。
    - 如果内容较长，则确立为“钩子 + Thread”的形式，并规划 Thread 的大致长度（例如 5-8 条推文）。

### 第 2 步：创作钩子推文 (Hook Tweet)

这是整个 Thread 最关键的一步，必须极具吸引力。

- **撰写文案**:
    - 提出一个大胆的、颠覆性的或极具价值的观点作为钩子。
    - 用 `A thread 🧵👇` 或类似标识明确告知读者这是一个 Thread。
- **提醒用户配图**:
    - **明确告知用户**: “第一条推文（钩子）必须配一张高质量的图片或 GIF 动图才能吸引注意力。请您准备一张最能代表本文核心思想的视觉材料。”
    - 在脚本中用占位符标出：`[钩子推文配图：请在此处添加一张图片/GIF]`

### 第 3 步：解构内容为 Thread

- **编号**: 将源内容拆解成一系列独立的知识点，并为每条推文编号 (`1/N`, `2/N` ...)。
- **一推一事**: 确保每条推文只讲一个核心论点，并保持语言简洁、有力、自信。
- **代码截图提醒**: 如果推文中需要包含代码，**提醒用户需要手动截图**，并在脚本中用占位符标出 `[代码截图：请在此处插入代码截图]`。

### 第 4 步：撰写结尾推文 (CTA Tweet)

最后一条推文是流量转化的关键。

- **添加链接**: 在推文末尾，明确引导用户访问外部链接，例如：“想深入了解？请查看我的完整博客文章：[链接地址]”。提醒用户替换链接。
- **添加 Hashtag**: 附上 3-5 个最相关的 Hashtag，例如 `#WebDev`, `#PWA`, `#Developer`。
- **鼓励互动**: 可以加上一句“你怎么看？”或“欢迎在下面讨论”来鼓励互动。

### 第 5 步：整合并调用 `code-docs-creator` 输出

- **整合内容**: 将以上所有推文（钩子、Thread、结尾 CTA）按顺序整合到一个 Markdown 文件中，形成一份完整的“发布脚本”。每条推文之间用分割线 `---` 隔开。
- **添加日期**: 在文档开头，明确标注“生成日期”，并使用当前系统日期。
- **确定文件名**: 以参考文章原文件名为主干，并追加平台引文简称后缀，例如 `[原文文件名]-x版.md`。
- **调用 `code-docs-creator`**:
    - 激活 `code-docs-creator` 技能。
    - 指示 `code-docs-creator` 将整合后的“脚本”内容，以上述文件名完成输出。
- **最终确认**: 告诉用户“推文脚本”已生成，并提供文件路径。
