# img-canvas-design 使用场景说明

简要说明 img-canvas-design 技能在前端开发工作流中的可落地用途, 作为独立学习资料, 不影响技能本体的执行流程。

生成日期: 2026-01-19
更新日期: 2026-01-19

## 1. 产品品牌与主视觉
- 目的: 通过抽象艺术建立品牌气质, 强化 Landing Page 首屏冲击力与差异化。
- 适配内容: 产品核心理念, 例如"速度" "稳定" "可信" "安全", 用结构化图谱表现。
- 输出建议: 1 张 A4 或 4:5 海报, 搭配 1-2 行极简标题, 可作为 Hero 背景或品牌封面。

## 2. 设计系统与组件库的视觉宣言
- 目的: 将 Design Tokens 或组件逻辑转译为图谱式艺术, 形成系统美学说明。
- 适配内容: 色彩分层, 间距尺度, 组件层级, 状态流转, 以线条与节点表达结构。
- 输出建议: 1 张"系统蓝图"海报 + 1 份哲学说明, 适合放入 README 或设计系统介绍页。

## 3. 技术博客与内容封面系列
- 目的: 为每篇文章生成系列封面, 保持统一风格与识别度。
- 适配内容: 文章主题的隐性线索, 例如"编译" "性能" "交互" "架构", 用几何与节奏暗示概念。
- 输出建议: 4:5 或 16:9 封面, 统一栅格与字体, 形成连贯系列。

## 4. 活动与发布宣发海报
- 目的: 让活动视觉更有质感, 避免模板化营销图。
- 适配内容: Hackathon, 内部分享, 产品发布, 用科学档案或抽象地图形式呈现信息。
- 输出建议: 1 张高密度视觉海报 + 1 行活动标题 + 1 行时间地点, 用于公众号或宣传页。

## 5. 个人作品集的章节页
- 目的: 以"视觉章节"提升作品集专业感, 展示审美与表达能力。
- 适配内容: 每个项目的核心概念, 作为抽象封面或章节分割页。
- 输出建议: A4 竖版或 16:10 横版, 用作作品集章节起始页。

## 6. 产品路线图与愿景表达
- 目的: 用艺术化图谱表达产品阶段性路线或未来愿景, 适合对外传播。
- 适配内容: 阶段里程碑, 版本代号, 核心能力演进, 用路径与节点隐喻。
- 输出建议: 1 张 A3 或 A4 竖版, 结合极简标签, 用于分享会或官网愿景页。

## 7. 数据叙事与指标主题视觉
- 目的: 将关键指标或数据故事转换为可视化艺术, 增强故事感。
- 适配内容: 留存, 转化, 负载, 体验稳定性, 用密度与节奏暗示变化。
- 输出建议: 1 张横版视觉, 可作为季度复盘或报告封面。

## 8. 开源项目视觉识别
- 目的: 为开源仓库建立稳定识别的视觉系统。
- 适配内容: 项目愿景, 架构理念, 社区精神, 用抽象语言形成"符号感"。
- 输出建议: 1 张主视觉 + 2-3 张系列图, 用于 README, Release Banner, 社交分享图。

## 使用建议
- 每次输出要求包含"哲学说明 + 视觉作品"组合, 方便持续复用与风格统一。
- 将主题转化为"隐性线索", 避免字面化表达, 让视觉成为叙事主体。
- 保持字体与栅格一致, 系列化输出可快速构建品牌与内容资产。
