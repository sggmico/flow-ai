# Code Master 补充说明

## 适用场景

- 用户要求“标准化检查/流水线/工程标准化/交付前审查/修正清单”
- 用户提供现有实现，希望得到可执行的修正清单
- 用户要求输出 Commit 信息、PR 描述或发布/回滚说明

## 示例（格式示意）

修正清单（Checklist）
- `src/auth/token.ts`：替换隐式 any 类型为显式 TokenPayload；补充 token 过期分支处理；验证：单测覆盖过期与缺失字段场景
- `src/api/user.ts`：统一错误处理为 AppError；补充 404 映射；验证：集成测试覆盖 404/500

是否通过标准化检查：FAIL
失败原因与阻断点：存在跨层依赖（service 直接访问 db），违反分层规则
