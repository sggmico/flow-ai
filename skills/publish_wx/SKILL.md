# 微信公众号文章发布助手

## 1. 核心目标

本技能旨在辅助将一篇技术文章（源文件）转化为符合微信公众号平台特性的格式，重点优化移动端阅读体验和社交传播性。

最终输出应是一份可以直接复制到微信编辑器进行二次排版的 Markdown 文本。

## 2. 工作流程

请严格遵循以下步骤完成文章转化任务。

### 第 1 步：分析源文章

- 读取用户提供的源文章内容，理解其核心思想、主要结构和关键知识点。

### 第 2 步：生成适配微信的标题

- **目标**: 创作一个有吸引力的、感性的、能引发读者好奇心的标题。
- **操作**:
    1.  基于源文章内容，构思 2-3 个符合微信风格的新标题。
    2.  可以采用“引人入胜的主标题 + 解释性的副标题”的结构。
    3.  向用户展示构思的标题，并请求用户选择或提出修改意见。

### 第 3 步：转化正文内容

这是最核心的步骤。你需要将正文转化为“扫读友好”和“视觉化”的格式。

- **语言风格**:
    - 将书面语调整为更口语化、更亲切的语言。
    - 拆分长句和长段落，保持每个段落只有一到三句话。

- **视觉化处理 (关键步骤)**:
    1.  **识别代码块**: 找出文章中所有的代码块。
    2.  **提醒用户截图**: **你无法直接转化代码块**。请明确告知用户：“微信公众号不支持直接复制代码块，为了保证最佳阅读效果，请您使用 [carbon.now.sh](http://carbon.now.sh) 或类似工具将以下代码块手动截图，并替换掉文本版本。”
    3.  在输出的 Markdown 中，用明确的占位符标记需要替换为图片的位置，例如：`[请在这里插入 <代码块 1> 的截图]`。
    4.  **识别复杂逻辑**: 找出文章中适合用图表表达的复杂逻辑、流程或核心观点。
    5.  **建议用户制图**: 向用户建议：“这部分内容如果能制作成流程图或卡片图，会更易于读者理解和分享。” 并在相应位置留下占位符，例如：`[建议：此处可插入 <核心架构> 的流程图]`。

- **排版优化**:
    - 在关键段落或句子前，适当添加 Emoji (如 💡, ✅, 🎯) 来吸引注意力。
    - 对核心概念或结论使用**粗体**进行强调。

### 第 4 步：撰写分享摘要

- 根据文章核心内容，撰写一段 50-100 字的摘要。
- **目标**: 这段摘要将用于公众号的分享卡片，需要足够吸引人，能够促使用户在社交网络中点击阅读。

### 第 5 步：添加标准结尾

- 在文章末尾，加入标准的行动号召 (Call to Action) 组合，引导读者进行互动和关注。
- **标准结尾模板**:
    ```markdown
    ---
    感谢阅读！如果觉得这篇文章对你有帮助，别忘了**点赞**、**在看**、**分享**给你的朋友。

    *   **🤔 你怎么看？** 欢迎在评论区留下你的想法！
    *   **🔗 深度阅读**: [点击此处阅读原文](链接到你的 GitHub 博客文章)
    *   **👋 保持联系**: 关注我的公众号「**你的公众号名称**」，获取最新技术干货。
    ```
- **提醒用户**: 请提醒用户将模板中的“链接到你的 GitHub 博客文章”和“你的公众号名称”替换为他们自己的信息。

### 第 6 步：整合并调用 `code_docs-creator` 输出

- **整合内容**: 将以上所有部分（标题、正文、摘要、结尾）整合为最终的 Markdown 文本。
- **添加日期**: 在文档开头，明确标注“生成日期”，并使用当前系统日期。
- **确定文件名**: 为文章生成一个符合 `code_docs-creator` 规范的中文文件名，例如 `微信公众号文章_{文章主题}.md`。
- **调用 `code_docs-creator`**:
    - 激活 `code_docs-creator` 技能。
    - 指示 `code_docs-creator` 将整合后的 Markdown 内容，以上述文件名，写入到默认的 `docs/` 目录下。
    - **示例指令**: "请使用 `code_docs-creator`，将以下内容保存到 `docs/微信公众号文章_如何学习Go语言.md` 文件中。"
- **最终确认**: 告诉用户文章已生成，并提供文件路径。
