# X (Twitter) 推文发布核心规范

本规范旨在将一个技术主题或长篇文章，转化为适合在 X (Twitter) 平台进行高效传播和引流的“推文脚本”。

## 核心原则

- **钩子为王**: 第一条推文 (Hook) 决定了整个 Thread 的成败。必须在 2 秒内抓住眼球。
- **信息碎片化**: 长篇内容必须被拆解成一系列短小、精悍、独立的推文。
- **视觉优先**: 钩子推文必须配图（最好是 GIF 或信息图），其他推文中的代码也应以截图形式呈现。
- **引流是最终目的**: Thread 的结尾必须将流量导向博客、产品等外部链接。

## 格式与风格要点

### 1. 整体结构

- **钩子 + Thread**: 由 1 条钩子推文和 N 条编号的系列推文组成。

### 2. 钩子推文 (Hook Tweet)

- **风格**: 简短、有力、充满悬念或提出一个大胆的观点。
- **结构**: 一句话引出话题，并用 `🧵👇` 或 `A thread...` 明确告知这是一个 Thread。
- **视觉**: **必须**配一张吸引人的图片、GIF 或图表。

### 3. 系列推文 (Thread Tweets)

- **编号**: 每条推文都应以 `1/N`, `2/N` 的形式编号，方便读者追踪。
- **一推一事**: 每条推文只讲一个核心论点或步骤。保持简洁。
- **代码截图**: 如果需要展示代码，应使用截图，不要直接粘贴文本。

### 4. 结尾推文 (Final Tweet)

- **Hashtag**: 在最后一条推文中，集中添加 3-5 个核心标签，如 `#100DaysOfCode`, `#WebDev`, `#PWA` 等。
- **行动号召 (CTA)**:
    - **附上链接**: 明确附上指向博客文章、项目源码或产品的链接。
    - **鼓励互动**: 鼓励用户“转发 (Retweet)”或“引用 (Quote)”来分享自己的看法。
