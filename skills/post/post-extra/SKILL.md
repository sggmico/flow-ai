# Generate Companion Content

为主文章生成结构化的配套文档，涵盖清单、行动建议、挑战、学习路径与资源。

## 快速开始

输入主文章内容（可选原始上下文），输出完整 Markdown 配套文档。

## 输入

- `MAIN_ARTICLE_CONTENT`：主文章完整内容。
- `ORIGINAL_CONTEXT`：可选上下文，用于补充工具和资源。

## 输出

仅输出 Markdown，必须包含以下固定章节（顺序不可变）：
1. 安全检查清单
2. 行动建议
3. 实战挑战
4. 互动讨论
5. 学习路径
6. 社区资源
7. 工具清单
8. 参考资源
9. 声明/版权/作者/下篇预告

## 执行流程

1. 解析主文章主题与技术栈。
2. 若提供上下文，提取额外工具与资源。
3. 按固定模板补全所有章节（不可省略）。
4. 输出原始 Markdown，不添加解释。

## 模板（必须遵循）

```markdown
# [文章标题] - 扩展信息

> 配套主文档：《[文章标题]》

---

## 安全检查清单

...

## 行动建议

...

## 实战挑战

...

## 互动讨论

...

## 学习路径

...

## 社区资源

...

## 工具清单

...

## 参考资源

...

## 声明/版权/作者/下篇预告

...
```

## 注意事项

- 若章节不适用，填写“本主题不适用”，但保留标题。
- 内容具体、可执行，避免空泛描述。
