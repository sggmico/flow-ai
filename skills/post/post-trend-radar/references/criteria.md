# 赛道权重与传播价值评估标准

## 赛道权重

| 赛道 | 权重 | 说明 |
|------|------|------|
| AI/LLM/Agent | 5 | 当前最高关注度，传播性最强 |
| Web3/区块链 | 4 | 技术社区高度关注，有投资属性 |
| 前端/Web | 4 | 开发者基数大，易于理解和试用 |
| 开发者效率工具 | 4 | 刚需强，转化率高 |
| Infra/DevOps | 3 | 专业性强，受众相对窄 |
| 游戏/个人项目 | 1 | 传播价值有限 |

---

## 传播价值判断 (1-5 分)

按以下 4 个问题快速评分，取平均值:

### 1. 痛点强度
- **5 分**: 明显省时省钱，能替代旧方案，刚需
- **4 分**: 解决明确问题，有清晰使用场景
- **3 分**: 有一定用处，但非必需
- **2 分**: 边角功能，受众窄
- **1 分**: 无明确痛点

### 2. 颠覆性/新范式
- **5 分**: 让用户不再写某类代码，改变工作方式
- **4 分**: 明显比现有方案优秀
- **3 分**: 渐进式改进
- **2 分**: 现有方案的变体
- **1 分**: 无新意

### 3. 易理解
- **5 分**: 一句话能讲清，非技术人也能懂
- **4 分**: 5 分钟能讲清
- **3 分**: 需要技术背景才能理解
- **2 分**: 需要深入了解才能明白价值
- **1 分**: 极难解释

### 4. 可试用
- **5 分**: 有在线 Demo，一键体验
- **4 分**: 本地可快速运行 (< 5 分钟)
- **3 分**: 需要一定配置才能运行
- **2 分**: 安装复杂，依赖多
- **1 分**: 无法试用或闭源

## 媒体传播潜力分级

| 等级 | 标准 | 适合平台 |
|------|------|----------|
| **High** | 一句话能解释"新范式/新能力"，有 Demo | 全平台爆款潜力 |
| **Medium** | 对特定人群很有用，门槛略高 | 公众号/X 深度文 |
| **Low** | 偏内部工程/边角工具 | 技术博客补充 |

## 信号过滤规则

### 正向信号 (加分)

| 信号 | 加分 | 说明 |
|------|------|------|
| 最近 7 天有实质提交 | +1.0 | 活跃维护 |
| README 清晰 | +0.5 | 一句话定位 + 示例/截图 |
| 许可证明确 (MIT/Apache) | +0.5 | 可放心使用 |
| 有 CI/workflow | +0.3 | 工程化程度高 |
| issue/discussion 活跃 | +0.5 | 社区认可 |
| release 有节奏 | +0.3 | 持续迭代 |
| 有 hacktoberfest/good-first-issue | +0.3 | 社区友好 |

### 负向信号 (减分或剔除)

| 信号 | 减分 | 说明 |
|------|------|------|
| 只有"Star me"营销 | -2.0 | 无实质内容 |
| Star 暴涨但活跃度低 | -1.5 | 疑似刷量 |
| README 过度承诺 | -1.0 | 缺少 demo/边界说明 |
| 名称含 "awesome-/list-" | -1.0 | 聚合类非原创 |
| 明显灰黑产/恶意用途 | 剔除 | 安全风险 |

## 评分公式

```
最终分 = 热度分 × (赛道权重/5) × 传播价值 + 信号调整

热度分 = log10(stars) × 2 + 活跃度加成 (0-10)
传播价值 = (痛点 + 颠覆 + 易理解 + 可试用) / 4 (1-5)
信号调整 = 正向信号加分 - 负向信号减分
```

## 竞品差异化原则

不做"项目搬运"，每个推荐必须回答:

1. **为什么现在火** - 技术趋势背景
2. **属于哪条技术路线** - 赛道定位
3. **是否值得投入时间** - 明确判断

竞品对比公式:
```
竞品 = GitHub 热度 × 可读性
我们 = GitHub 热度 × 技术趋势 × 决策价值
```
