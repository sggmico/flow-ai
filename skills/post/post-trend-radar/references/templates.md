# 文章模板与平台适配

## 一、榜单型主文 (公众号)

### 结构

```markdown
# 本周 GitHub 趋势观察 | {主题关键词}

> 数据口径: 截至 {日期}，时间窗口: {window}

## 趋势引子

{2-3 句交代赛道变化，建立"为什么现在要看这个"的 hook}

例如:
"这周 AI Agent 赛道继续爆发，3 个项目同时冲上 GitHub Trending 前 10。
更值得注意的是，它们都在解决同一个问题: 让 LLM 能自主完成多步骤任务。"

---

## 01 {项目名}

**一句话价值定位**: {简洁描述核心价值}

{技术解释 + 应用场景，2-3 段}
- 段落 1: 这是什么，解决什么问题
- 段落 2: 技术特点，为什么比其他方案好
- 段落 3: 适合谁用，有什么限制

**GitHub 数据**:
- Stars: {count} (↑{growth} 近 7 天)
- 语言: {language}
- 许可: {license}
- 链接: {url}

---

## 02 {项目名}
...

---

## 本周技术趋势总结

### 哪条技术路线在加速?
{观察和判断}

### 哪类开发者应该关注?
{具体建议}

### 我最看好的是:
**{项目名}** - {判断理由}

---

> 关注我，每周获取 GitHub 技术趋势解读
```

### 要点

- 标题要有关键词: "本周"/"GitHub"/"趋势"
- 开场 hook 要强: 建立"为什么现在"
- 每个项目要有一句话定位
- 必须有判断观点，不只是搬运

## 三、平台适配版本

### 小红书版

**特点**: 生活化、视觉化、碎片化

```markdown
标题: {emoji} {生活化标题}

例如:
🔥 程序员必看！这个 AI 工具让我效率翻倍
💡 不会写代码也能用的 GitHub 神器

---

正文:

{emoji} 发现一个超实用的工具!

要点 1: {简短描述}
要点 2: {简短描述}
要点 3: {简短描述}
要点 4: {简短描述}
要点 5: {简短描述}

{emoji} 总结: {一句话}

{emoji} 链接在评论区~

---

标签:
#程序员 #效率工具 #GitHub #开源 #{相关标签}
```

**注意**:
- 标题必须有 emoji
- 正文分点，每点不超过 20 字
- 总字数控制在 300-500 字
- 标签 5-8 个

---

**特点**: 简洁、有 hook、线程展开

```markdown
主帖 (280 字内):

{hook} + {核心价值}

{emoji} {关键特点}

GitHub: {short_url}

🧵 详细介绍 ↓

---

线程:

1/ {项目是什么}
{一句话描述}

2/ {解决什么问题}
{痛点描述}

3/ {为什么现在火}
{背景分析}

4/ {我的判断}
{观点}
```

**注意**:
- 主帖要有强 hook
- 每条线程独立完整
- 3-5 条线程为宜
- 最后一条带 CTA

## 四、文章风格指南

### 技术深度风
- 重点: 原理解析、实现细节
- 适合: 公众号技术文、博客
- 篇幅: 2000-4000 字

### 趋势解读风
- 重点: 行业判断、发展预测
- 适合: 公众号、X 线程
- 篇幅: 1500-2500 字

### 工具推荐风
- 重点: 实用指南、快速上手
- 适合: 全平台
- 篇幅: 1000-2000 字

### 媒体爆款风
- 重点: hook 强、易传播
- 适合: 小红书、B 站
- 篇幅: 500-1500 字

## 五、输出文件命名规范

遵循 code-docs-creator skill 规则:

```
{NNN}_{主题}.md            # 主文 (公众号)
{NNN}_{主题}_小红书.md      # 小红书版
{NNN}_{主题}_X.md          # X 版
{NNN}_{主题}_B站.md        # B 站版
```

- `{NNN}`: 三位数序号，如 001, 002
- `{主题}`: 中文优先，简洁明了
- 多版本输出在末尾标明版本

例如:
```
001_AI-Agent周报.md
001_AI-Agent周报_小红书.md
001_AI-Agent周报_X.md
001_AI-Agent周报_B站.md
```

默认输出目录: `/Users/sggmico/course/post/`
