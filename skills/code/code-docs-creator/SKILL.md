# 文档输出规范


## 文档规范（必须遵循）

- 格式：Markdown，标题层级清晰，段落紧凑。
- 语言：中文优先；必要时补充英文术语。
- 命名：`001_主题.md` 格式，中文优先。
- 标点：默认 ASCII 标点。
- 日期：所有文档必须标注生成/更新日期，必须使用系统实际日期，不得猜测或编造；若无法获取日期，先询问用户。
- 测试：测试用例描述中文优先，必要时补充英文函数名。

## 输出位置（必须遵循）

- 先确认一级目录：默认目录或指定目录。
- 默认目录: 当前项目的 `docs/`。
- 若指定目录: 使用用户指定目录并确认路径。
- 确认一级目录后，询问是否以当前使用的生成模型名创建子目录，作为最终输出目录。
- 若指定目录，先查看该目录已有文档的格式；其格式优先级最高，须优先对齐。



## 文档类型变体（择一）

- README：标题、摘要、特性、安装/启动、使用方式、配置、排错、贡献、许可
- 设计/规格说明：标题、摘要、问题、范围、需求、设计、API/数据、测试、发布/回滚、风险
- 指南/SOP：标题、目的、前置条件、步骤、验证、回滚、FAQ
- 报告：标题、摘要、方法、发现、证据、建议、下一步
- 发布说明：标题、摘要、新增、变更、修复、弃用、破坏性变更
- API 文档：标题、摘要、认证、接口、请求/响应、错误、示例

## 输出规则
- 以清晰标题与简短摘要开头。
- 在文档内显式标注“生成日期 / 更新日期”。
